Class ydn.api.User

Providing authentication for user, use object can be used as simple object or as presenter (in MVP). Both client session and server session are used.

Constructor
ydn.api.User()
Create a user object.
API user do not initiate user object, but acquired from application instance using getUser() method. 
var app = new ydn.api.App(options);
  var user = app.getUser();

Events
{string} user-loaded
User data is loaded from the local Storage.
{string} user-logging-in
User initiate login.
{string} user-login
Received login data from the server.
{string} user-logging-out
User initiate logout.
{string} user-logout
Received logout data from the server.

Methods
addEventListener(type, handler, capture)
Adds an event listener to the this storage object.
The same handler can only be added once per the type. Even if you add the same handler multiple times using the same type then it will only be called once when the event is dispatched.
Parameters:
{string} type
event type.
{Function|Object} handler
The function to handle the event. The handler can also be an object that implements the handleEvent method which takes the event object as argument.
{boolean=} capture
whether the listener is fired during the capture or bubble phase of the event.
getNickname()
Get nickname set by the server.
Returns:
{string|undefined}Nickname.
getEmail()
Get email of current login user.
Returns:
{string|undefined}User email.
getLoginUrl()
Get login url set from the login server.
Returns:
{string|undefined}Login url if user is not login.
isLogin()
Check login status.
Returns:
{boolean}true if user is login.
getLogoutUrl()
Get logout url set from the login server.
Returns:
{string|undefined}Logout url if user is login. OpenID do not have logout url. Simply clear client session.
removeEventListener(type, handler, capture)
Removes an event listener from the event target.
The handler must be the same object as the one added. If the handler has not been added then nothing is done.
Parameters:
{string} type
event type.
{Function|Object} handler
The function to handle the event. The handler can also be an object that implements the handleEvent method which takes the event object as argument.
{boolean=} capture
whether the listener is fired during the capture or bubble phase of the event.
